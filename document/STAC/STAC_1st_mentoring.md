## 디자인 피드백

```markdown
- 일단 Web의 UI가 굉장히 깔끔한 편이다. 조금의 수정을 가한다면 충분히 출품하기에 적합하다(특히 직관적인 UI가 보기좋다)
- Web 카드에 내용을 좌로 정렬하고 Icon형식을 이용해 카테고리를 분류한다면, 더욱 UI가 부각될 것이다
- 팀의 테마색이 있다면 Web에 조금 더 입혀도 좋을 것 같다. 그리고 그에 맞게 배경색과 카드의 그림자를 디자이너의 느낌에 맞게 조절한다면, 더 좋은 UI를 지닐 것이다
- Android Application UI에서 장애인을 위한 UX와 정상인을 위한 UX가 잘 어우러져 굉장히 디자인이 좋다. 추가로 시각장애인을 위한 고대비를 사용한다면 더 장애인들이 보기 좋을 것 같다. 시각 장애인 분들이 전부 완전히 안보이는 것이 아니기 때문에 만약 시력이 조금이라도 있다면 고대비가 더 보기 편할 것이다.
- 대부분의 UI가 높은 직관성을 띄고 있어서 디자이너가 UI를 열심히 구성한 것 같다는 느낌이다.
* Logo의 형태가 조금 더 정사각형에 가까울 필요가 있다. 로고의 지팡이가 직사각형의 느낌이 강해 이쁘긴 하지만, 비율에 따라 크게 디자인적으로 차이가 많이 날 것 같다는 의견이다. google guide-line을 본다면 정사각형에 가깝게 만들기 때문에 그것을 참고하면 더 좋을 것 같다. 또한 Logo에서 폰트를 좀 더 신경쓴다면 Logo가 더욱 아름답게 바뀔 것이다
- 모듈의 케이스를 아무래도 제작하는 것이 좋을 것 같다. 일반인들이 모듈, 그 자체를 쓴다면 굉장한 거부감을 느낄 것이다. 부착하기에 편리하도록 케이스를 제작한다면 팀과 소비자 모두 좋을 것이다
```

AppJam의 하루간 결과물로써는 굉장히 퀄리티가 높고 아이디어도 좋다. 또한 개발과 디자인마저 꽤 잘되어 있어서 놀랐다. 다른 팀들은 이제 막 시작하는 분위기이지만, 이 팀은 이미 꽤 많은 단계가 되어 있고 기획적인 부분에서도 조금만 더 하게된다면 여유를 가지고 작업을 진행할 것 같다. 멘토링을 이제 2번 더 해야하는 데 디자인 멘토이다 보니 디자이너만 와도 괜찮을 것 같다. 9월 말이 최종 마감이니 디자인 멘토링은 개발이 급하다면 8월초와 8월말에 2번하여 마무리 짓고, 만약 개발단이 완벽하다면 8월에 한번 9월 초에 한번하여 완성도를 높이면 될 것 같다. 현재 첫번째 멘토링인데도 많이 진행되어 있으니 남은 잔 작업을 마무리 져서 다음 멘토링을 하면 될 것 같다. 디자인적으로 고민이 있을 때는 언제든 물어봐도 좋답니당 ^____^

## 멘토링 후 든 내 생각

```markdown 
우리가 우선시 해야할 점은 기획의 정확성과 시각 장애인이 사용하기 편리한 UX를 잡아가는 것이라고 생각한다. 이것을 확실히 한 후에 단계적으로 개발과 나머지 디자인을 잡아가는 것이 맞는 것 같다. 또한 일반인들도 거부감 없이 사용할 수 있도록 이 솔루션을 만드는 것이 매출을 위한다면 좋을 것 같다. 물건을 조금 자주 잃어버리는 일반인은 사용할 법한 솔루션이라는 생각이 강하다. 그러기 위해서는 조금 더 장애인과 일반인의 디바이드 없이 접점의 UI를 구성해야할 것 같다. 이 점은 디자이너(내)가 많은 고민이 필요할 것 같다.(할 일 생겨서 마음이 아프다;;) 개발측에서는 기획과 디자인쪽의 세세한 요구를 잘 녹여내야 할 것 같다. 특히 장애인을 위한 UX적인 측면이 해당된다.
```
